# 变更日志

> **目的**：记录 PRD 和设计方案的所有变更，以及变更带来的连锁影响。
>
> **与 evolution.md 的区别**：
> - `evolution.md` — 开发进度日志，按 Phase 记录「做了什么」
> - `CHANGELOG.md` — 需求和设计变更记录，记录「改了需求/设计 + 为什么 + 影响了什么」

---

## 变更记录格式

每条变更包含：

| 字段 | 说明 |
|------|------|
| **ID** | `CHG-001` 递增 |
| **日期** | YYYY-MM-DD HH:mm |
| **变更类型** | PRD 变更 / 设计变更 / 两者都有 |
| **触发原因** | 什么导致了这个变更（用户反馈/技术限制发现/业务方调整/...） |
| **变更内容** | 改了什么 |
| **影响分析** | 哪些文件/模块/Step 需要同步修改 |
| **同步检查** | 逐项确认受影响的部分已更新 |
| **关联** | 链接到 evolution.md 的相关条目 |

---

## 设计同步协议（必须遵守）

> 当 PRD 或设计文档发生变更时，**必须按以下顺序完成同步**，不允许跳过。

```
PRD 变更
  │
  ├── 1. 记录到 CHANGELOG.md（本条）
  ├── 2. 检查设计影响：
  │   ├── API 接口需要变更？ → 更新 design.md
  │   ├── DB Schema 需要变更？ → 生成新 migration
  │   ├── 前端组件需要变更？ → 更新对应 Step
  │   └── 都不需要 → 说明原因
  ├── 3. 检查验收场景影响：
  │   └── acceptance.md 的 Given-When-Then 需要增减？
  ├── 4. 检查下游 Step 影响：
  │   └── 已完成的 Step 需要回滚？未开始的 Step 需要重新拆解？
  └── 5. 追加 evolution.md：
      └── 记录：「YYYY-MM-DD HH:mm | 变更 | CHG-xxx - <变更摘要>」
```
