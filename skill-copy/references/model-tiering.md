# 模型分层策略

## 三层模型架构

| 层级 | 适用场景 | 默认模型 | 备选模型 | 实现方式 |
|------|---------|---------|---------|----------|
| **T1** | 测试用例生成、文档生成、格式化、linting | Claude Haiku | Qwen-Turbo / GPT-4o-mini | Subagent 执行 |
| **T2** | 业务逻辑、CRUD、API、重构、测试修复（1-2次） | Claude Sonnet | DeepSeek-V3 / GPT-4o | 主 Agent 直接执行 |
| **T3** | 核心算法、安全逻辑、支付、系统架构、测试修复（第3次） | Claude Opus | GPT-4o / Claude Sonnet+人工 | 主 Agent 深度推理 / Subagent |

## 模型单价参考

| 模型 | 输入/1M tokens | 输出/1M tokens |
|------|--------------|---------------|
| Claude Haiku | $0.25 | $1.25 |

## 实现方式说明

| 层级 | 执行方式 | 说明 |
|------|---------|------|
| T1 | Subagent 执行 | 主 Agent 启动一个独立的、配置为 T1 模型的 Subagent 来执行任务（测试生成、文档生成）。Subagent 返回结果后主 Agent 继续。 |
| T2 | 主 Agent 直接执行 | 主 Agent 自身使用默认或配置的 T2 模型执行。不需要启动独立 Subagent。 |
| T3 | 主 Agent 深度推理 / Subagent | 复杂任务由主 Agent 深度推理完成。安全/支付等高风险任务可启动独立 Subagent 确保隔离。 |
## 路由决策树

```
用户需求
├─ 架构设计/系统设计 → T3
├─ 任务拆解（复杂场景） → T2/T3

├─ 测试用例生成 → T1
├─ 文档生成/更新 → T1
├─ 代码格式化/lint → T1
├─ CRUD/API/表单 → T2
├─ 业务逻辑/规则 → T2
├─ 代码重构 → T2
├─ 测试修复（第1-2次） → T2
├─ 测试修复（第3次/复杂错误） → T3
├─ 安全审查/代码 Review → T3
├─ 支付/计费逻辑 → T3
├─ 认证/权限/安全 → T3
├─ 系统架构/设计决策 → T3
├─ 数据库迁移/表结构变更 → T3
└─ 不确定 → T2
```

## 成本控制规则

1. 严格按路由表选择模型，**不准越级升级**（禁止 T1 应处理的任务使用 T2，T2 能处理的用 T3）
2. **允许降级**：T3 可降级到 T2，T2 可降级到 T1（适用于简单任务，节省成本）
3. 每次升级（T1→T2 或 T2→T3）必须在 `.vibecoding/logs/model-upgrades.log` 记录原因
4. 链式升级路径：`T1 → T2 → T3`，不允许跳级（如 T1 直接到 T3）

## 降级策略参考

以下场景可以主动降低模型层级以节省成本：

| 场景 | 降级路径 | 节省 |
|------|---------|------|
| 任务极其简单（如常量定义） | T2→T1 | ~90% |
| 只是格式化或重排代码 | T2→T1 | ~90% |
| 用户要求快速原型 | 全链路 T1 | ~80% |

## Token 预算参考（月度）

基于 30 个功能点/月：
- T1: ~1M tokens → ~$0.3-1.0
- T2: ~3M tokens → ~$9.0-15.0
- T3: ~0.3M tokens → ~$4.5-22.5
- **总计: ~$14-38/月**（全用 Sonnet 约 $60，全用 Opus 约 $300）
